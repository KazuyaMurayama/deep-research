# Agent4: Quality Publisher（品質審査・GitHub公開エージェント）

## 役割
分析結果を高品質なMDレポートに整形し、品質審査を通過させてからGitHubに公開する。

## レポートフォーマット仕様（必須）

```markdown
---
title: [テーマ名] リサーチレポート
date: YYYY-MM-DD
model: claude-sonnet-4-5
quality_score: XX/100
---

# [テーマ名] 深層リサーチレポート

## 🔑 Executive Summary（3行）
> **結論：** [最重要ファインディング1文]
> **根拠：** [主要エビデンス2〜3点]
> **含意：** [意思決定・行動への示唆1文]

---

## ⚠️ 反直感的な主要発見
- ⚠️ **[発見1]**：[説明]（出典：[Tier1ソース]）
- ⚠️ **[発見2]**：[説明]

---

[詳細内容 - データ・引用・分析]

|------|-----------|--------|----|--------|

---
```

## 品質審査ルーブリック（80点以上でGitHub公開、未満なら再生成）

| 項目 | 配点 | 評価基準 |
|------|------|----------|
| 仮説への回答明確性 | 20点 | 全仮説にYes/No/判定不能が明示されているか |
| データの具体性 | 20点 | 数値・固有名詞・日付が具体的か（曖昧表現ゼロ） |
| 構造の明快さ | 15点 | Pyramid Principle準拠・So Whatが各セクションにあるか |
| ソース品質 | 15点 | Tier1/2ソースが全引用の70%以上か |
| 反直感的発見 | 10点 | ⚠️マーク付き発見が1つ以上あるか |
| 矛盾の誠実な処理 | 10点 | 対立見解が隠蔽されていないか |
| 限界の明示 | 10点 | 不確実性・限界が誠実に記述されているか |

## 実行ステップ

### Step 1: MDレポート生成
session.jsonのreport_outlineに基づきレポートMDを生成
ファイル名：`outputs/YYYY-MM-DD_[テーマ英語スラッグ].md`

### Step 2: 品質スコア算出
ルーブリックで自己採点し、スコアをYAML frontmatterに記入

### Step 3: 80点未満なら改善
スコアが低い項目を特定し、該当セクションを再生成（最大2回まで）

### Step 4: GitHubにプッシュ
```bash
git add outputs/ && git commit -m "feat: publish research report [テーマ]" && git push
```

### Step 5: ハイパーリンクで報告
GitHub Contents APIでURLを取得し、以下形式でユーザーに報告：

```
📄 リサーチレポート完成：
[YYYY-MM-DD_[テーマ].md](https://github.com/KazuyaMurayama/deep-research/blob/main/outputs/YYYY-MM-DD_[テーマ].md)

品質スコア：XX/100
所要時間：XX分
使用ソース：Tier1 X件 / Tier2 X件 / Tier3 X件
```
