## 2. なぜ「ただ録音するだけ」ではダメなのか

### 2.1 両声キャプチャ問題の本質

オンライン会議の音声は「2つの音声ストリーム」が同時に存在します。

```
あなたの声（マイク入力）  →  Razer Seiren Mini  →  PCに入力
相手の声（スピーカー出力）→  スピーカーから出力 →  PCから出力
```

通常の録音ソフトで「マイクを録音」すると、**あなたの声しか録れません**。単一指向性マイクはさらに徹底して周囲の音を排除するため、相手の声が入る余地すらありません。

```
❌ 失敗するやり方
スマートフォンのICレコーダーアプリでZoom会議を録音
→ スピーカーから出る相手の声を物理的に拾うが、音質が悪い

❌ さらに失敗するやり方
Windows 標準ボイスレコーダーでマイク録音
→ 自分の声のみ。相手の発言がまるごと消える
```

```
✅ 正しいやり方：システム音声（スピーカー出力）を仮想的にキャプチャ
相手の声（デスクトップ音声）＋自分の声（マイク）を合成して録音
```

### 2.2 Windows 11特有の問題

Windowsには「ステレオミキサー」という機能がありますが、**USB接続のRazer Seiren Miniを使っている場合は動作しないことが多い**です。

| 接続方式 | ステレオミキサー | WASAPI Loopback |
|---------|----------------|-----------------|
| 3.5mmジャック（アナログ） | 多くの場合使用可能 | 使用可能 |
| USB（Razer Seiren Mini等） | ほぼ使用不可 | 使用可能 ✅ |

> 💡 **WASAPI（ウォサピ）とは**: Windows Audio Session API の略。Windowsが提供するオーディオ処理の低レイヤーAPI。「Loopback」モードでスピーカーに出力される音声を仮想的に取得できる。

### 2.3 各プラットフォームが問題をどう解決するか

| 方式 | 解決方法 | メリット | デメリット |
|------|---------|---------|---------|
| プラットフォーム内蔵 | サーバー側で全音声を処理 | 設定不要 | 有料プラン必須 |
| Bot参加型SaaS | Botが会議に参加して録音 | 設定簡単 | クラウドにデータ送信 |
| OBS + Whisper | デスクトップ音声を仮想ルーティング | 完全ローカル | 初期設定が複雑 |

## 4. 音声キャプチャ方式の比較

### 4.1 方式A：プラットフォーム内蔵文字起こし

```
会議プラットフォーム（Zoom/Teams/Meet）
│
├── 参加者A（あなた）の音声 ─→ サーバーで合成・文字起こし
│
└── 参加者B（相手）の音声 ─→ サーバーで合成・文字起こし
```

**仕組み**: 会議プラットフォームのサーバーが全参加者の音声を処理するため、ローカルでの音声キャプチャが不要。

| 項目 | Zoom | Teams | Google Meet |
|------|------|-------|------------|
| 文字起こし機能 | AI Companion（有料） | 会議の文字起こし（有料） | Meet文字起こし（有料） |
| 必要プラン | Zoom Pro以上（$16/月〜） | M365 Business Basic以上 | Google Workspace有料 |
| 日本語対応 | 2024年から対応 | 対応 | 2025年3月から正式対応 |
| 話者分離 | ✅ | ✅ | ✅ |
| エクスポート | テキスト/VTT | テキスト/Word | テキスト/SRT |

### 4.2 方式B：Bot参加型SaaS

```
あなたの会議URL
│
├── あなた（参加者）
├── 相手（参加者）
└── Notta Bot / AiNote Bot（仮想参加者として録音・文字起こし）
```

**仕組み**: 文字起こしサービスのBotが会議に「参加者」として加わり、すべての音声を録音・処理。

| サービス | 無料枠 | 有料プラン | 日本語精度 | 特徴 |
|---------|--------|---------|---------|------|
| Notta | 月120分 | Premium: 月1,185円〜 | 高（日本語特化） | UIが日本語 |
| LINE WORKS AiNote | 月300分 | Solo: 月1,440円 | 最高（CLOVA技術） | 話者分離が特に優秀 |
| tl;dv | 月10会議 | Pro: $29/月 | 中（英語圏ツール） | Chrome拡張で手軽 |
| Fireflies.ai | 月800分 | Pro: $18/月 | 中（日本語は二次対応） | 英語が主軸 |

### 4.3 方式C：OBS + Whisperローカル処理

```
デスクトップ音声（相手の声） ──→ OBS（録音ソフト）──→ 音声ファイル（MP3/WAV）
マイク音声（自分の声）      ──→ OBS（録音ソフト）──→          │
                                                              ↓
                                                    Whisper（AI文字起こし）
                                                              │
                                                              ↓
                                                    テキストファイル
```

> 💡 **OBS（オービーエス）とは**: Open Broadcaster Software の略。もともと配信用ソフトだが、デスクトップ音声とマイクを同時に録音できるため、議事録用途にも使える。完全無料。
>
> 💡 **Whisper（ウィスパー）とは**: OpenAIが開発した音声認識AI。特に日本語精度が高く、large-v3モデルは現時点で最高レベルの精度。

**OBS + Whisperの特長**:
- ✅ 完全ローカル処理（音声データがクラウドに送られない）
- ✅ 最高精度の日本語文字起こし（Whisper large-v3）
- ✅ 完全無料
- ✅ Zoom/Teams/Meet問わず対応
- ❌ 初期設定に30〜60分必要
- ❌ Pythonの基礎知識が必要

### 5.2 各ツール詳細

#### Notta
- **概要**: 日本語に特化したAI文字起こしSaaS。Bot参加でZoom/Teams/Meet対応。
- **強み**: UIが完全日本語対応。話者分離機能あり。エクスポート形式が豊富（TXT/DOCX/SRT/PDF）。
- **弱み**: 無料枠は月120分と少ない。長時間会議が多い場合は課金が必要。
- **Claude Code連携**: TXT/DOCXでエクスポート後にコピペ。フォーマットがきれいで分析しやすい。

#### LINE WORKS AiNote（旧CLOVA Note）
- **概要**: 2025年7月に個人向け正式リリース。韓国NAVER社のCLOVA技術を使用。話者分離は世界トップレベル（2024年競合評価3位）。
- **強み**: 話者分離が特に高精度。日本語固有名詞も正確。月300分の無料枠はSaaSの中で最大級。
- **弱み**: LINE WORKS アカウントが必要。Soloプランで月1,440円。
- **Claude Code連携**: テキストエクスポート後にコピペ。話者ラベルが明確なためClaude Codeでの分析が容易。

#### OBS（Open Broadcaster Software）
- **概要**: 映像配信・録画ソフト。バージョン28以降でWASAPI Loopback（システム音声）録音に対応。
- **設定のポイント**:
  - 音声ソース1: WASAPI デスクトップ音声（相手の声）
  - 音声ソース2: マイク（Razer Seiren Mini）
  - 出力: MP4/MKV形式で保存
- **公式サイト**: https://obsproject.com/

#### Whisper
- **概要**: OpenAI製の音声認識モデル。ローカルPCで動作するため、音声データがクラウドに送られない。
- **モデル選択**:

| モデル | 精度 | 処理速度 | VRAM目安 |
|-------|------|---------|---------|
| tiny | 低 | 非常に速い | 1GB |
| base | 中 | 速い | 1GB |
| small | 中〜高 | 普通 | 2GB |
| medium | 高 | やや遅い | 5GB |
| **large-v3** | **最高** | 遅い | **10GB** |

> ⚠️ large-v3モデルはGPUがない場合、1時間の音声を文字起こしするのに数十分かかることがあります。GPUがある場合は数分で完了します。

## 7. セットアップ手順書（Windows 11対応）

---

### 🟢 シナリオA セットアップ：Zoom内蔵文字起こし

**所要時間**: 5分
**前提**: Zoom Proアカウント（$16/月以上）

#### 手順

**Step 1: 設定から文字起こしを有効化（2分）**

1. Zoom Webポータル（https://zoom.us/profile/setting）にログイン
2. 「会議」タブ → 「文字起こし」を検索
3. 「クラウドレコーディングの自動文字起こし」をONにする
4. 「AI Companion」の「会議の文字起こし」をONにする

> ⚠️ 無料プランでは文字起こし機能は使用できません。

**Step 2: 会議中に文字起こしを開始（1分）**

1. 会議を開始
2. 画面下部のツールバーから「文字起こし」ボタンをクリック
3. 「会議の文字起こし」を選択
4. 全参加者の画面に「文字起こし中」と表示される

**Step 3: 会議後にダウンロード（1分）**

1. Zoom Webポータル → 「レコーディング」→「クラウドレコーディング」
2. 該当会議を選択
3. 「テキスト文字起こし（VTT形式）」をダウンロード
4. または「Audio Transcript（TXT形式）」をダウンロード

> 💡 TXT形式のほうがClaude Codeへの貼り付けがしやすいです。

**Step 4: Claude Codeへ貼り付け（1分）**

ダウンロードしたTXTファイルをテキストエディタで開き、内容をコピーしてClaude Codeに貼り付けます。

---

### 🟡 シナリオB セットアップ：OBS + Whisperローカル処理

**所要時間**: 初回30〜60分
**難易度**: 中級（Pythonの基礎知識があると良い）
**コスト**: 完全無料

#### 7.B.1 OBSインストールと設定（15〜20分）

**Step 1: OBSをダウンロード・インストール（5分）**

1. https://obsproject.com/ にアクセス
2. 「Windows」ボタンをクリック
3. ダウンロードしたインストーラーを実行
4. デフォルト設定でインストール完了

**Step 2: 初回起動設定（5分）**

1. OBSを起動
2. 「自動構成ウィザード」が表示されたら「キャンセル」を押す（録画専用設定をするため）
3. 「設定」→「出力」→「出力モード: 詳細」に変更
4. 「録画」タブで以下を設定:
   - 録画フォーマット: `mp4`
   - エンコーダ: `x264`（GPUがなければ）または `NVENC`（NVIDIA GPU）
   - 録画パス: 任意のフォルダ（例: `C:\Users\[ユーザー名]\Documents\MeetingRecordings`）

**Step 3: 音声ソースを設定（10分）**

> ⚠️ これが最重要設定です。両声キャプチャを実現します。

1. 「ソース」パネルの「+」ボタンをクリック
2. 「音声入力キャプチャ」を選択 → 名前: `マイク（Razer Seiren Mini）`
3. デバイス: `Razer Seiren Mini` を選択 → OK
4. 再度「+」ボタンをクリック
5. 「音声出力キャプチャ（WASAPI）」を選択 → 名前: `デスクトップ音声（相手の声）`
6. デバイス: 現在使用しているスピーカーまたはヘッドフォン出力を選択 → OK

> 💡 **WASAPI（ウォサピ）とは**: Windows Audio Session API の略。Windowsが提供する音声処理APIで、スピーカーに出力される音声を「録音可能な入力」として扱える機能（Loopbackモード）を持っています。

**Step 4: 音声ミキサーで確認（5分）**

1. 「音声ミキサー」パネルを確認
2. 自分がマイクに向かって話すと「マイク」バーが動くことを確認
3. 動画（YouTube等）を再生すると「デスクトップ音声」バーが動くことを確認

> ⚠️ ヘッドフォン使用時の注意: Bluetooth/USB接続のヘッドフォンでは「音声出力キャプチャ（WASAPI）」が使えない場合があります。その場合は次の「VB-Audio Cable」手順を参照してください。

#### 7.B.3 録音から文字起こしまでのパイプライン（5分）

**Step 1: OBSで会議を録画**

2. `Ctrl+Alt+R` で録画開始（またはツールバーの「録画開始」ボタン）
3. 会議終了後 `Ctrl+Alt+R` で録画停止
4. ファイルが `C:\Users\[ユーザー名]\Documents\MeetingRecordings\` に保存される

**Step 2: Whisperで文字起こし実行**

```powershell
# 基本的な文字起こし（日本語）
whisper "C:\Users\[ユーザー名]\Documents\MeetingRecordings\[ファイル名].mp4" --language Japanese --model medium

# 高精度版（large-v3モデル、時間がかかる）
whisper "C:\Users\[ユーザー名]\Documents\MeetingRecordings\[ファイル名].mp4" --language Japanese --model large-v3

# 出力形式をテキストのみに指定
whisper "C:\Users\[ユーザー名]\Documents\MeetingRecordings\[ファイル名].mp4" --language Japanese --model medium --output_format txt
```

> 💡 初回実行時はモデルのダウンロード（数百MB〜数GB）が発生します。2回目以降はキャッシュされるため速いです。

**Step 3: 出力確認**

文字起こし結果は同じフォルダに `.txt` ファイルとして保存されます。

**一括処理スクリプト（自動化用）**

会議後に毎回コマンドを打つのが手間な場合は、以下のPowerShellスクリプトを保存しておくと便利です:

```powershell
# transcribe.ps1 として保存
param(
    [string]$InputFile = "",
    [string]$Model = "medium"
)

if ($InputFile -eq "") {
    # 最新のMP4ファイルを自動取得
    $InputFile = Get-ChildItem "C:\Users\$env:USERNAME\Documents\MeetingRecordings\*.mp4" | 
                 Sort-Object LastWriteTime -Descending | 
                 Select-Object -First 1 -ExpandProperty FullName
}

Write-Host "文字起こし開始: $InputFile"
whisper $InputFile --language Japanese --model $Model --output_format txt
Write-Host "完了しました。"
```

実行方法:
```powershell
# 最新の録画ファイルを自動処理
.\transcribe.ps1

# ファイルを指定して処理
.\transcribe.ps1 -InputFile "C:\path\to\meeting.mp4" -Model large-v3
```

## 10. トラブルシューティング

### 10.1 音声関連の問題

| 症状 | 原因 | 解決策 |
|------|------|--------|
| 相手の声が録音されない | デスクトップ音声ソースが未設定 | OBSで「音声出力キャプチャ（WASAPI）」を追加 |
| 自分の声が録音されない | マイクソースが未設定 or デバイスが違う | OBSの「音声入力キャプチャ」でRazer Seiren Miniを選択 |
| 音声が全く入らない | OBSの音声ミキサーがミュート | ミキサーパネルのミュートボタン（🔇）を確認 |
| ヘッドフォンから音が出なくなった | VB-Audio Cable設定後 | サウンド設定で出力デバイスを元のヘッドフォンに戻す |
| 音声が小さすぎる | マイクゲインが低い | OBSの音声ミキサーでゲインを調整（右クリック → フィルタ → ゲイン追加） |

### 10.2 Whisper関連の問題

| 症状 | 原因 | 解決策 |
|------|------|--------|
| `whisper` コマンドが見つからない | インストール失敗 or PATHが未設定 | `pip install openai-whisper` 再実行 → PowerShell再起動 |
| `ffmpeg` が見つからない | ffmpeg未インストール | `winget install ffmpeg` 実行 → 再起動 |
| 文字起こし精度が低い | モデルが小さい | `--model medium` または `--model large-v3` オプションを追加 |
| 処理が非常に遅い | CPUのみで処理中 | GPU対応版: `pip install faster-whisper` + CUDAインストール |
| 日本語が英語に変換される | 言語指定忘れ | `--language Japanese` オプションを必ず追加 |
| メモリ不足エラー | large-v3モデルがRAM不足 | `--model medium` に変更 |

**Whisper再インストールコマンド（リセット用）**:

## 12. Typelessの正しい活用方法

### 12.1 Typelessとは何か（正確な理解）

**Typeless**は「口述入力ツール」です。キーボードの代わりに声でテキストを入力するためのサービスであり、**会議の全参加者の声を録音・文字起こしする用途には対応していません**。

| 用途 | Typelessで可能か |
|------|----------------|
| 自分の声をリアルタイムでテキスト入力 | ✅ 可能（主要用途） |
| 会議中の自分の発言だけを記録 | ✅ 可能（限定的） |
| 相手の声を含む会議全体の録音 | ❌ **不可能** |
| Zoom/Teamsの会議音声を文字起こし | ❌ **不可能** |

> ⚠️ **重要**: 「Typelessで会議の両声録音ができる」という情報は**誤り**です。Typelessはマイクからの音声入力専用ツールであり、スピーカーから出力される相手の声をキャプチャする機能を持っていません。

### 12.2 Typelessが活きる場面

1. **会議中の自分のメモ入力**: キーボードを打つ音がうるさい場合、声でメモを入力
2. **会議後の感想・所感を音声入力**: 会議終了直後に感想を音声で記録
3. **議事録の口述作成**: 文字起こしツールで得たテキストをもとに、自分の口述でまとめる

### 12.3 Typeless × 他ツールの組み合わせ例

```
[シナリオ] 会議の文字起こし + 自分の補足メモを追加したい

1. OBS + Whisperで会議全体を文字起こし（両声対応）
2. 会議後、Typelessを起動
3. 文字起こし結果を見ながら、補足・訂正・感想を音声入力
4. 完成した議事録+メモをClaude Codeで分析
```
