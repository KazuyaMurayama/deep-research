## 2. なぜ「ただ録音するだけ」ではダメなのか

### 2.1 両声キャプチャ問題の本質

オンライン会議の音声は「2つの音声ストリーム」が同時に存在します。

```
あなたの声（マイク入力）  →  Razer Seiren Mini  →  PCに入力
相手の声（スピーカー出力）→  スピーカーから出力 →  PCから出力
```

通常の録音ソフトで「マイクを録音」すると、**あなたの声しか録れません**。単一指向性マイクはさらに徹底して周囲の音を排除するため、相手の声が入る余地すらありません。

```
❌ 失敗するやり方
スマートフォンのICレコーダーアプリでZoom会議を録音
→ スピーカーから出る相手の声を物理的に拾うが、音質が悪い

Windows 標準ボイスレコーダーでマイク録音
→ 自分の声のみ。相手の発言がまるごと消える

✅ 正しいやり方：システム音声（スピーカー出力）を仮想的にキャプチャ
→ 相手の声（デスクトップ音声）＋自分の声（マイク）を合成して録音
```

### 2.2 Windows 11特有の問題

Windowsには「ステレオミキサー」という機能がありますが、**USB接続のRazer Seiren Miniを使っている場合は動作しないことが多い**です。

| 接続方式 | ステレオミキサー | WASAPI Loopback |
|---------|----------------|-----------------|
| 3.5mmジャック（アナログ） | 多くの場合使用可能 | 使用可能 |
| USB（Razer Seiren Mini等） | ほぼ使用不可 | 使用可能 ✅ |

> 💡 **WASAPI（ウォサピ）とは**: Windows Audio Session API の略。Windowsが提供するオーディオ処理の低レイヤーAPI。「Loopback」モードでスピーカーに出力される音声を仮想的に取得できる。

### 2.3 各プラットフォームが問題をどう解決するか

| 方式 | 解決方法 | メリット | デメリット |
|------|---------|---------|---------|
| プラットフォーム内蔵 | サーバー側で全音声を処理 | 設定不要 | 有料プラン必須 |
| Bot参加型SaaS | Botが会議に参加して録音 | 設定簡単 | クラウドにデータ送信 |
| OBS + Whisper | デスクトップ音声を仮想ルーティング | 完全ローカル | 初期設定が複雑 |

> ⚠️ **録音の同意取得について**: 日本では会議参加者全員の同意なしに録音することは法的にグレーな場合があります。会議録音を行う前に、参加者全員への事前通知と同意取得を強く推奨します（本ガイド全体に適用）。

## 4. 音声キャプチャ方式の概要

### 方式A：プラットフォーム内蔵文字起こし

会議プラットフォームのサーバーが全参加者の音声を処理するため、ローカルでの音声キャプチャが不要。

| 項目 | Zoom | Teams | Google Meet |
|------|------|-------|------------|
| 文字起こし機能 | AI Companion（有料） | 会議の文字起こし（有料） | Meet文字起こし（有料） |
| 必要プラン | Zoom Pro（$13.33/月・年払い / $16.99/月・月払い） | M365 Business Basic以上 | Google Workspace有料 |
| 日本語対応 | 2024年から対応 | 対応 | 2025年3月から正式対応 |
| 話者分離 | ✅ | ✅ | ✅ |

### 方式B：Bot参加型SaaS

文字起こしサービスのBotが会議に「参加者」として加わり、すべての音声を録音・処理。

### 方式C：OBS + Whisperローカル処理

デスクトップ音声とマイク音声をOBSで合成録音し、Whisperでオフライン文字起こし。

- ✅ 完全ローカル処理（音声データがクラウドに送られない）
- ✅ 最高精度の日本語文字起こし（Whisper large-v3）
- ✅ 完全無料・Zoom/Teams/Meet問わず対応
- ❌ 初期設定に30〜60分、Pythonの基礎知識が必要

### 5.2 各ツール詳細

#### Notta
- **概要**: 日本語に特化したAI文字起こしSaaS。Bot参加でZoom/Teams/Meet対応。
- **強み**: UIが完全日本語対応。話者分離機能あり。エクスポート形式が豊富（TXT/DOCX/SRT/PDF）。
- **弱み**: 無料枠は月120分と少ない。長時間会議が多い場合は課金が必要。

#### LINE WORKS AiNote（旧CLOVA Note）
- **概要**: 2025年8月に正式リリース（前身のCLOVA Note βは2025年7月31日終了）。韓国NAVER社のCLOVA技術を使用。話者分離はDIHARD3チャレンジ（2021年）でNAVERチームが上位入賞（LINE WORKS公式）するなど世界トップレベル。
- **強み**: 話者分離が特に高精度。日本語固有名詞も正確。月300分の無料枠はSaaSの中で最大級。
- **弱み**: LINE WORKS アカウントが必要。Soloプランで月1,440円。

#### OBS（Open Broadcaster Software）
- **概要**: 映像配信・録画ソフト。バージョン28以降でWASAPI Loopback（システム音声）録音に対応。
- **設定のポイント**:
  - 音声ソース1: WASAPI デスクトップ音声（相手の声）
  - 音声ソース2: マイク（Razer Seiren Mini）
  - 出力: MP4/MKV形式で保存
- **公式サイト**: https://obsproject.com/

#### Whisper
- **概要**: OpenAI製の音声認識モデル。ローカルPCで動作するため、音声データがクラウドに送られない。
- **モデル選択**:

| モデル | 精度 | 処理速度 | VRAM目安 |
|-------|------|---------|---------|
| tiny | 低 | 非常に速い | 1GB |
| base | 中 | 速い | 1GB |
| small | 中〜高 | 普通 | 2GB |
| medium | 高 | やや遅い | 5GB |
| **large-v3** | **最高** | 遅い | **推奨10GB（int8量子化利用で8GB以下でも動作可能）** |

> ⚠️ large-v3モデルはGPUがない場合、1時間の音声を文字起こしするのに数十分かかることがあります。GPUがある場合は数分で完了します。

### 🟡 シナリオB セットアップ：OBS + Whisperローカル処理

**所要時間**: 初回30〜60分
**難易度**: 中級（Pythonの基礎知識があると良い）
**コスト**: 完全無料

#### 7.B.1 OBSインストールと設定（15〜20分）

**Step 1: OBSをダウンロード・インストール（5分）**

1. https://obsproject.com/ にアクセス
2. 「Windows」ボタンをクリック
3. ダウンロードしたインストーラーを実行
4. デフォルト設定でインストール完了

**Step 2: 初回起動設定（5分）**

1. OBSを起動
2. 「自動構成ウィザード」が表示されたら「キャンセル」を押す（録画専用設定をするため）
3. 「設定」→「出力」→「出力モード: 詳細」に変更
4. 「録画」タブで以下を設定:
   - 録画フォーマット: `mp4`
   - エンコーダ: `x264`（GPUがなければ）または `NVENC`（NVIDIA GPU）
   - 録画パス: 任意のフォルダ（例: `C:\Users\[ユーザー名]\Documents\MeetingRecordings`）

**Step 3: 音声ソースを設定（10分）**

> ⚠️ これが最重要設定です。両声キャプチャを実現します。

1. 「ソース」パネルの「+」ボタンをクリック
2. 「音声入力キャプチャ」を選択 → 名前: `マイク（Razer Seiren Mini）`
3. デバイス: `Razer Seiren Mini` を選択 → OK
4. 再度「+」ボタンをクリック
5. 「音声出力キャプチャ（WASAPI）」を選択 → 名前: `デスクトップ音声（相手の声）`
6. デバイス: 現在使用しているスピーカーまたはヘッドフォン出力を選択 → OK

> 💡 **WASAPI（ウォサピ）とは**: Windows Audio Session API の略。スピーカーに出力される音声を「録音可能な入力」として扱えるLoopbackモードを持つWindowsのオーディオAPI。

**Step 4: 音声ミキサーで確認（5分）**

1. 「音声ミキサー」パネルを確認
2. 自分がマイクに向かって話すと「マイク」バーが動くことを確認
3. 動画（YouTube等）を再生すると「デスクトップ音声」バーが動くことを確認

> ⚠️ ヘッドフォン使用時の注意: Bluetooth/USB接続のヘッドフォンでは「音声出力キャプチャ（WASAPI）」が使えない場合があります。その場合は以下の「VB-Audio Cable」手順を参照してください。

**（オプション）VB-Audio Cableのインストール**

1. https://vb-audio.com/Cable/ からVB-Audio Virtual Cable（無料）をダウンロード
2. インストール後、Windowsの「サウンド設定」→「出力デバイス」を「VB-Audio Virtual Cable」に変更
3. OBSの「音声出力キャプチャ」デバイスも「VB-Audio Virtual Cable」に変更
4. Zoomの「スピーカー設定」も「VB-Audio Virtual Cable」に設定

> ⚠️ VB-Audio Cable設定後は、通常のスピーカー/ヘッドフォンから音が聞こえなくなります。会議後は出力デバイスを元に戻すか、「Audio Router」等で両方同時出力する設定が必要です。

#### 7.B.2 Whisperインストール（15〜20分）

**Step 1: Pythonのインストール確認（3分）**

PowerShellを開いて以下を実行:

```powershell
python --version
```

`Python 3.9`以上が表示されれば OK。表示されない場合は https://www.python.org/downloads/ からインストール。

> 💡 Pythonインストール時は「Add Python to PATH」のチェックボックスを必ずONにしてください。**PATHとは**: Windowsが「このコマンドはどこにあるプログラムを実行するか」を探す場所のリスト。ONにするとどのフォルダからでも `python` コマンドが使えるようになります。

**Step 2: Whisperのインストール（5〜10分）**

```powershell
pip install openai-whisper
```

> ⚠️ エラーが出た場合は `pip install git+https://github.com/openai/whisper.git` を試してください。

**（オプション）faster-whisperのインストール（高速版）**

処理速度を2〜4倍に改善したい場合:

```powershell
pip install faster-whisper
```

**Step 3: ffmpegのインストール（5〜10分）**

WhisperはMP4などの動画ファイルを処理するためにffmpegが必要です。

```powershell
# winget（Windows Package Manager）でインストール
winget install -e --id Gyan.FFmpeg
```

インストール後、PowerShellを再起動してから確認:

```powershell
ffmpeg -version
```

#### 7.B.3 録音から文字起こしまでのパイプライン（5分）

**Step 1: OBSで会議を録画**

1. OBSを起動（会議開始前）
2. ツールバーの「録画開始」ボタンをクリック（Settings > Hotkeysで任意のキーを割り当て可能）
3. 会議終了後、ツールバーの「録画停止」ボタンをクリック
4. ファイルが `C:\Users\[ユーザー名]\Documents\MeetingRecordings\` に保存される

**Step 2: Whisperで文字起こし実行**

PowerShellで以下を実行:

```powershell
# 基本的な文字起こし（日本語）
whisper "C:\Users\[ユーザー名]\Documents\MeetingRecordings\[ファイル名].mp4" --language Japanese --model medium

# 高精度版（large-v3モデル、時間がかかる）
whisper "C:\Users\[ユーザー名]\Documents\MeetingRecordings\[ファイル名].mp4" --language Japanese --model large-v3

# 出力形式をテキストのみに指定
whisper "C:\Users\[ユーザー名]\Documents\MeetingRecordings\[ファイル名].mp4" --language Japanese --model medium --output_format txt
```

> 💡 初回実行時はモデルのダウンロード（数百MB〜数GB）が発生します。2回目以降はキャッシュされるため速いです。

**Step 3: 出力確認**

文字起こし結果は同じフォルダに `.txt` ファイルとして保存されます。

**一括処理スクリプト（自動化用）**

```powershell
# transcribe.ps1 として保存
param(
    [string]$InputFile = "",
    [string]$Model = "medium"
)

if ($InputFile -eq "") {
    # 最新のMP4ファイルを自動取得
    $InputFile = Get-ChildItem "C:\Users\$env:USERNAME\Documents\MeetingRecordings\*.mp4" | 
                 Sort-Object LastWriteTime -Descending | 
                 Select-Object -First 1 -ExpandProperty FullName
}

Write-Host "文字起こし開始: $InputFile"
whisper $InputFile --language Japanese --model $Model --output_format txt
Write-Host "完了しました。"
```

実行方法:
```powershell
# 最新の録画ファイルを自動処理
.\transcribe.ps1

# ファイルを指定して処理
.\transcribe.ps1 -InputFile "C:\path\to\meeting.mp4" -Model large-v3
```

## 7. Claude Code連携：議事録テンプレート・プロンプト集

### 7.1 推奨入力フォーマット

Claude Codeへ貼り付ける際は、以下のフォーマットに整えると分析精度が大幅に向上します。

```markdown
# 議事録
- 日時: 2026-MM-DD HH:MM-HH:MM（例: 2026-05-20 14:00-15:00）
- 参加者: 自分 / 相手A / 相手B
- テーマ: （例: 新プロジェクト キックオフ）
- ツール: （例: Zoom + LINE WORKS AiNote）

## 本文（話者別・タイムスタンプ付き）
### 自分
(00:00) おはようございます。本日はお時間をいただきありがとうございます。

### 相手A
(01:15) よろしくお願いします。先日ご連絡いただいた件ですね。
```

> 💡 話者ラベルを「自分」「相手A」「相手B」と統一しておくと、以下のプロンプトテンプレートをそのまま使いまわせます。また、Claude Codeへ貼り付ける際、個人名を「Aさん」「Bさん」に置換する習慣をつけるとプライバシーリスクを低減できます。

### 7.2 プロンプト集

#### プロンプト1：会議要約（最頻用途）

```
以下の会議議事録を分析して、次の4ブロックで再整理してください:

(1) 3行要約（箇条書き）
(2) 決定事項（決定内容・理由・決定者を含む）
(3) アクションアイテム（担当者・期限・タスク内容を含む表形式）
(4) 未解決論点（次回会議で確認すべき事項）

---
[ここに議事録テキストを貼り付け]
```

#### プロンプト2：タスク抽出

```
以下の議事録から、アクションアイテムのみを抽出してください。
出力形式: `- [ ] 担当者 / 期限 / タスク内容`

---
[ここに議事録テキストを貼り付け]
```

#### プロンプト3：話者ラベル正規化

```
以下の議事録で、話者ラベルの表記揺れを統一してください。
- 「私」「自分」「田中」→ 「自分」に統一
- 「相手」「Aさん」「山田さん」→ 「相手A」に統一

統一後の全文をそのまま出力してください。

---
[ここに議事録テキストを貼り付け]
```

#### プロンプト4：新メンバー向けFAQ化

```
以下の会議議事録から、新しくプロジェクトに参加するメンバーが理解すべき情報を
FAQ形式（5問）で作成してください。

各問は次の形式で:
Q: （質問）
A: （回答・100字以内）
出典: （タイムスタンプまたは発言者）

---
[ここに議事録テキストを貼り付け]
```

#### プロンプト5：リスク抽出

```
以下の会議議事録から、リスク・懸念事項・反対意見をすべて抽出してください。

出力形式:
| リスク内容 | 発言者 | リスクレベル（高/中/低） | 対応案 |
|----------|--------|---------------------|------|

リスクレベルの判定基準:
- 高: プロジェクト継続に関わる重大な問題
- 中: 対応が必要だが解決可能
- 低: 軽微な懸念・注意事項

---
[ここに議事録テキストを貼り付け]
```

#### プロンプト6：メール下書き自動生成

```
以下の会議議事録をもとに、参加者への「会議後フォローアップメール」の下書きを作成してください。

メール要件:
- 宛先: [相手の名前・会社名をここに入力]
- 件名は「【会議御礼】[テーマ]」形式
- 内容: 要点3行 + 決定事項 + 自分のアクションアイテム + 次回会議の確認依頼

---
[ここに議事録テキストを貼り付け]
```

[発言内容]

## 8. セキュリティ・プライバシー指針

### 8.1 機密度別ツール選択

```
会議の機密度はどのくらいですか？
│
├── [非常に高い] 社外秘・個人情報・M&A情報等
│   └── ✅ シナリオB（OBS + Whisperローカル処理）一択
│       ※音声・テキストが外部サービスに送信されない
│
├── [中程度] 社内情報・プロジェクト情報
│   └── [会社がSaaS使用を認めているか？]
│       ├── YES → ✅ シナリオA（プラットフォーム内蔵）または シナリオC（Notta/AiNote）
│       └── NO  → ✅ シナリオB（OBS + Whisperローカル処理）
│
└── [低] 一般的な打ち合わせ・情報収集・勉強会
    └── ✅ どのシナリオでも可（コスト・使いやすさで選択）
```

### 8.3 個人情報保護の対応策

| 項目 | 対応策 |

## 9. トラブルシューティング

### 9.1 音声関連の問題

| 症状 | 原因 | 解決策 |
|------|------|--------|
| 相手の声が録音されない | デスクトップ音声ソースが未設定 | OBSで「音声出力キャプチャ（WASAPI）」を追加 |
| 自分の声が録音されない | マイクソースが未設定 or デバイスが違う | OBSの「音声入力キャプチャ」でRazer Seiren Miniを選択 |
| 音声が全く入らない | OBSの音声ミキサーがミュート | ミキサーパネルのミュートボタン（🔇）を確認 |
| ヘッドフォンから音が出なくなった | VB-Audio Cable設定後 | サウンド設定で出力デバイスを元のヘッドフォンに戻す |
| 音声が小さすぎる | マイクゲインが低い | OBSの音声ミキサーでゲインを調整（右クリック → フィルタ → ゲイン追加） |

### 9.2 Whisper関連の問題

| 症状 | 原因 | 解決策 |
|------|------|--------|
| `whisper` コマンドが見つからない | インストール失敗 or PATHが未設定 | `pip install openai-whisper` 再実行 → PowerShell再起動 |
| `ffmpeg` が見つからない | ffmpeg未インストール | `winget install -e --id Gyan.FFmpeg` 実行 → 再起動 |
| 文字起こし精度が低い | モデルが小さい | `--model medium` または `--model large-v3` オプションを追加 |
| 処理が非常に遅い | CPUのみで処理中 | GPU対応版: `pip install faster-whisper` + CUDAインストール |
| 日本語が英語に変換される | 言語指定忘れ | `--language Japanese` オプションを必ず追加 |
| メモリ不足エラー | large-v3モデルがRAM不足 | `--model medium` に変更、またはint8量子化（faster-whisper推奨） |

**Whisper再インストールコマンド（リセット用）**:

```powershell
pip uninstall openai-whisper
pip install openai-whisper
pip install setuptools-rust
```

### 9.3 Bot型SaaS（Notta/AiNote）の問題

| 症状 | 原因 | 解決策 |
|------|------|--------|
| Botが会議に参加しない | URL入力ミス or 連携設定未完了 | 会議URLを再確認 → OAuthを再認証 |
| 文字起こしが日本語にならない | 言語設定が英語のまま | アカウント設定 → 言語 → 日本語に変更 |
| 話者が区別されない | 参加者登録が未設定 | 設定 → 話者 → 参加者名を登録 |
| 無料枠が超過した | 月の利用時間を超えた | 別サービスに切り替えるか課金 |

### 9.4 Claude Codeへの貼り付けがうまくいかない場合

```
問題: テキストが大きすぎてClaude Codeのコンテキストに入らない
解決策:
1. 会議の前半・後半で分割して貼り付ける
2. まず「30分ずつ分割して貼り付けます」と伝えてから投入する
3. 要約プロンプトを使って先に要約を作成し、それをもとに分析する
```

**Bot参加に失敗した場合のフォールバック**:

```
1. 会議ツールの「クラウドレコーディング」機能で録音（音声ファイルを後処理）
2. 会議後に音声ファイルをWhisperに通す
3. または、会議ツールのURL共有でNotta等のBotに手動で録音リクエスト
```
